Integrating the Orders API: Reference Notes

These notes cover authentication, idempotency, and rate limits for teams integrating against the Orders backend. They are split across two pages for length.

1. Authentication

All requests authenticate with a backend API key passed in the X-Orders-Key header. Keys are environment-scoped; never log them.

2. Idempotency

Supply an Idempotency-Key on writes. Retries with the same key return the original result rather than duplicating the order.

3. Rate limits

Default 120 requests/minute per key, burst 240. Exceeding returns 429 with a Retry-After header.
